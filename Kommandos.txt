Voraussetzung: Account + Eingeladen zur Kollaboration

Zunächst:
    1. Git installieren:  git-scm.com
    2. Terminal (vorzugsweise Anaconda Prompt): git --version
    3. Terminal: git clone URL_DES_REPOSITORY -> URL des Repository einfügen (https://github.com/JohannesHaenle/Data-Mining-StarWars), dadurch wird Repository lokal geklont
    4. Terminal: cd NAME_DES_REPOSITORY -> in das Verzeichnis navigieren (Data-Mining-StarWars)
    5. Terminal: jupyter notebook -> öffnet Jupyter Notebook im Verzeichnis 

Änderungen speichern und pushen (von Lokal zu GitHub):
    wenn Änderungen lokal vorgenommen, müssen diese im Repository gespeichert werden
    Um Dateien im Repository von Lokal zu synchroniseren, müssen folgende Befehle nacheinander im Terminal ausgeführt werden.
    Wichtig: Zuvor ins lokale Verzeichnis navigieren (cd NAME_DES_REPOSITORY)
        git add .
        git commit -m "Erster Commit" -> "" durch beliebigen (am besten beschreibenden) Kommentar ersetzen
        git push origin main


    Falls nicht funktioniert, muss zu Beginn einmal Username und Mailadresse eingegeben werden. Hierzu folgende Befehle im Terminal und Verzeichnis des Repository ausführen 
    git config --global user.name "Ihr Name" -> "" ersetzen
    git config --global user.email "ihre.email@beispiel.com" -> "" ersetzen

Vom Repository das lokale Verzeichnis updaten (also umgekehrt, von GitHub zu Lokal):
    git pull origin main #Änderungen im Repository werden ins lokale Verzeichnis übernommen
